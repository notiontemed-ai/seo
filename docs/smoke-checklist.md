# Ручной smoke-чеклист (приёмка)

Автотесты не покрывают интеграцию с боевым Bitrix/n8n и браузерный UI — этот
чеклист прогоняется вручную после деплоя.

## Предусловия

- `local/api/seo/config.php`: заданы разные `read_token` и `write_token`.
- `internal/seo-editor/config.php`: `password_hash`, `temed_seo_api_url`,
  `temed_seo_api_token`, `temed_seo_api_write_token`, `n8n_base_url`, `n8n_secret`.
- Собран редактор: `cd internal/seo-editor/app && npm install && npm run build`.
- n8n workflow импортирован, секрет задан.

## Навигация (ТЗ v1.1)

0. **Шапка** — три раздела (Генерация контента / Аналитика / Администрирование) +
   кнопка «Ассистент» + переключатель отладки. Кнопок «Транскрибация» и «Черновики»
   в шапке нет. «Аналитика» и «Администрирование» открываются как пустые экраны с
   заголовком раздела.

## Основной сценарий редактора

1. **Вход** — открыть `/internal/seo-editor/`, авторизоваться паролем.
2. **Задача (шаг 1)** — секции сверху вниз: «Кейсовый вход (аудио)» (свёрнут, если
   бриф не пуст) со встроенной транскрибацией; «Интент и структура» — выбор интента
   показывает ровно 3 его структуры, выбор структуры открывает детали (чипы блоков,
   «Запрещено», версия); «Запросы» — primary + secondary чипами; «Атрибуты» —
   название, код (автотранслит), раздел, регион, автор, медредактор, связи. Справа
   «Сводка брифа» отражает готовность; в задаче НЕТ полей Анонс, Краткий ответ,
   Тип статьи, SEO title, Meta description.
3. **Генерация (шаг 2)** — «Сгенерировать черновик» → приходят блоки
   `article_content` v2; секция «Мета и анонс» предзаполнена (SEO title ≤60,
   Meta description ≤160, Анонс — со счётчиками) и редактируется; появляются вопросы
   медредактору; ответить и «Применить ответы врача».
4. **Текст (шаг 3)** — при выбранной структуре в пустом «Тексте» создаётся скелет
   обязательных блоков и работает чек-лист их готовности; добавить/удалить/
   переместить/сменить тип блока; «Предпросмотр» — статья рендерится (оглавление,
   FAQ, источники, дисклеймер), HTML не показывается.
5. **Проверки (шаг 4)** — «Каннибализация»: список кандидатов с risk/сигналами/
   фрагментами. TEXT.RU: «Запустить» → «Обновить статус».
6. **Перелинковка (шаг 5)** — «Найти доноров»: только активные доноры, светофор
   нагрузки, `found_links`; отметить галочками → «Скопировать ТЗ» / «Скачать .md».
7. **Публикация (шаг 6)** — «Опубликовать черновик» → модалка подтверждения
   (инфоблок 81, название, код, раздел, режим, ACTIVE=N) → «Подтвердить».

## Приёмка write-API (ключевое)

8. **Создание** — после публикации элемент появился в инфоблоке 81 **неактивным**
   (`ACTIVE=N`); ответ `created:true`, ссылка `admin_url` открывает карточку.
9. **Обновление** — изменить контент, опубликовать тем же кодом → тот же элемент
   обновлён (`updated:true`), остаётся неактивным.
10. **Защита активного** — в админке сделать элемент активным; повторить
    публикацию тем же кодом → ответ **409 `ACTIVE_ELEMENT_PROTECTED`**, элемент
    не изменился.

## Подсистемы

11. **Ассистент** (глобальная боковая панель) — задать вопрос; suggestion-карточка
    появляется только при валидном `field_id`+`value`; «Применить» меняет поле.
    «Поле: undefined» не возникает.
12. **Черновики** (вкладка раздела «Генерация контента», рядом с «Мастером») —
    индикатор автосохранения в панели раздела; «Сохранить текущий» → появляется в
    списке; «Восстановить» возвращает блоки и поля.
13. **Транскрибация** (внутри шага «Задача») — загрузить аудио → карточки тем с
    primary и secondary запросами; «Применить тему» заполняет поля задачи (с
    подсветкой) и добавляет блок «Случай из практики», без смены экрана.
14. **Структуры (перенос конфига, ТЗ 4.1)** — `proxy.php?action=article_structures`
    отдаёт 9 конфигов из `internal/seo-editor/data/`; в публичном API действие
    `article_structures` отсутствует; `system_manifest` отвечает без ошибок.
15. **Совместимость** — старые локальные сессии (без поля `section`) открываются в
    «Генерации контента» на прежнем шаге; старые облачные снапшоты восстанавливаются.

## Автотесты

```bash
bash tests/run.sh
```

PHP: `article_draft_writer` (allowlist, ACTIVE=N, защита активных, upsert),
`html_renderer_snapshot`, `article_content`, `cannibalization`, `linking`,
`api_modules`, `exporter`. JS (Vitest): `blocks`, `suggestions`, `htmlRenderer`.
