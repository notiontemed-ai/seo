# Перелинковка (Этап 4)

Действие `linking_candidates` (POST) ищет статьи-доноры для анкоров на
редактируемую (целевую) статью. Результат — данные для ТЗ контент-менеджеру
(что и где вставить). **Автоматическая правка доноров не выполняется.**

## Поток

```
browser → proxy.php → local/api/seo/index.php (POST linking_candidates)
```

## Вход

```json
{
  "action": "linking_candidates",
  "article": {
    "element_id": 123,
    "code": "back-pain",
    "name": "",
    "primary_query": "",
    "secondary_queries": ["", ""],
    "article_content": { "schema_version": "2.0", "blocks": [] }
  },
  "limit": 20,
  "refresh_corpus": false
}
```

`element_id`/`code` нужны, чтобы определить URL цели (для `already_linked`) и
исключить саму статью из доноров. Текст цели — из `article_content` или `text`.

## Логика

Доноры — **только АКТИВНЫЕ** элементы инфоблоков 81 и 68 (должны быть
опубликованы). По каждому донору:

- **релевантность** — те же лексические сигналы, что в отчёте каннибализации
  (`TextSignals`): `query_overlap`, `title_similarity`, `text_overlap`;
  взвешенная сумма, порог отсечения нерелевантных;
- **подсчёт текстовых ссылок** (`DonorLinkParser`) — `<a>` внутри `<p>` и
  ячеек таблиц (`<td>/<th>`). Не считаются: ссылки в списках; блоки, чей
  ближайший предыдущий заголовок/абзац матчится с
  `/рекоменду|читайте также|похожие|источник/i`; блок источников
  (`class*="sources"`);
- **светофор нагрузки** — `link_load: green (<3) | yellow (3–5) | red (>5)`;
- **already_linked** — донор уже ссылается на целевую статью (по URL/коду);
  такой донор помечается, но из выдачи не убирается;
- **место вставки** — абзац донора с максимальным лексическим пересечением с
  целевой статьёй; варианты анкора — из primary/secondary запросов цели
  (лучшие варианты можно догенерировать через n8n `suggest_anchor`).

## Ответ (по кандидату)

```json
{
  "element_id": 10, "source": "new", "name": "", "url": "", "admin_url": "",
  "relevance": 0.42,
  "signals": { "query_overlap": 0.5, "title_similarity": 0.3, "text_overlap": 0.2 },
  "link_load": "green",
  "links_in_text": 2,
  "found_links": [{ "anchor": "", "href": "" }],
  "already_linked": false,
  "insertion": {
    "paragraph_quote": "",
    "position_hint": "Абзац №3 по тексту донора",
    "overlap": 0.31,
    "anchor_options": ["", ""]
  }
}
```

`found_links` показывается редактору, чтобы была видна база светофора.

## suggest_anchor (n8n)

Действие `suggest_anchor` (n8n): модель получает абзац донора + запросы цели и
возвращает строго JSON `{"anchors": ["", "", ""]}` (1–3 варианта). Используется
для уточнения анкоров при формировании ТЗ.

## UI (этап 5)

Редактор отмечает кандидатов галочками → «Сформировать ТЗ» → markdown-документ
(копируемый + скачиваемый): по каждому донору — ссылка в админку, цитата абзаца
«до», текст «после» со вставленным `<a href>` и анкором, позиция. Запись в
доноров не реализуется.

## Тесты

`tests/php/linking_test.php` — подсчёт текстовых ссылок на фикстурах с
«Рекомендуем также», таблицами и источниками; светофор; `already_linked`;
только активные доноры; релевантность; выбор абзаца; самоисключение цели.
