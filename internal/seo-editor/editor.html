Материалы

Клиника и услуга

Выбранные карточки передаются в задание как фактический контекст TEMED. Генератор не должен придумывать параметры услуги или филиала.

Связанные материалы

Факты и правила TEMED

Критично. Реальные параметры процедур (число инъекций, длительность, наличие/отсутствие УЗИ и т. п.) генератор берёт только отсюда. Без этого поля он подставит среднерыночные цифры.

Внешние источники

n8n найдёт источники, проверит утверждения и вернёт список для подтверждения.

Встроенный ассистент

Автоматический контур. Ассистент подключён к веб-приложению: задание, структура, контекст и источники передаются сами. Готовый текст появится на шаге «Результат и проверка».
Запрос выполняется через защищённый серверный proxy.php.

Загрузка готового текста

Медредактура

Ассистент собирает из текста спорные формулировки и вопросы для врача. Врач отвечает, ответы возвращаются в текст точечно.

Вопросы для медредактора

Ответы врача

Меняются только согласованные формулировки — остальной текст не трогается.

Результат и проверка

Проверьте и отредактируйте всё, что уйдёт в Bitrix, затем запустите автопроверки и проверку уникальности. Ничего не публикуется автоматически.

Мета и служебные поля

Текст статьи

Меняются только указанные части — статья не генерируется заново без необходимости.

Уникальность текста

Проверяется нормализованный текст из заголовка, анонса, короткого ответа и HTML статьи. Медицинский смысл автоматически не меняется.

Сформируйте задание на вёрстку, примените результат — и выпустите статью через XML-экспорт. Реестр шаблонов загружается из API — сейчас доступен default.

Задание

Задание включает полный текущий текст статьи. Скопируйте его целиком во внешний ИИ-ассистент.
Вставьте сюда полный JSON-ответ внешнего ИИ. Нажмите «Проверить результат», затем «Применить результат вёрстки».
После проверки и применения поле detail_html заменит итоговый текст статьи, из которого формируется XML.
Результат вёрстки ещё не загружен.
После применения результат будет записан в итоговые поля статьи. Bitrix XML формируется из итоговой версии статьи. XML будет сформирован из применённой версии статьи.
Сценарий 1 (текущий). Система формирует XML, вы скачиваете его и импортируете в Bitrix вручную. Права записи не выдаются.
Сценарий 2 (будущий). n8n через отдельный write-API создаёт только неактивный черновик в инфоблоке 81.
